現在のブランチのコミット履歴を分析し、日本語のPull Requestを作成する。

ブランチが作業内容に適しているか事前に確認したい場合は `/branch` スキルを先に実行することを推奨する。

## 手順

### 1. 現状の確認

以下を並列で実行して現状を把握する：

- `git status`（未コミットの変更確認。`-uall` フラグは使用しない）
- `git branch --show-current`（現在のブランチ名）
- `git remote -v`（リモートの確認）
- `git log --oneline -10`（直近のコミット履歴）

mainブランチにいる場合は、PRを作成できない旨を伝えてスキルを終了する。

ブランチ名が確定したら以下を実行する：

- `git log main..HEAD --oneline`（mainからの差分コミット一覧）
- `git diff main...HEAD`（mainからの差分内容）
- `gh pr list --head <branch-name>`（同一ブランチのPRが既に存在するか確認）

既存のPRが見つかった場合はその旨を伝えてスキルを終了する。

### 2. PRの内容分析

コミット一覧と差分を分析して以下を決定する：

**PRタイトル：**

- 変更内容全体を端的に表す日本語のタイトル（70文字以内）
- Conventional Commits形式：`<type>: <内容>`
- typeはコミット群の主要な変更種別を使用

**PR説明文：**

以下の構成で生成する。
`[あなたのエージェント名]`には実行しているあなた自身（AIエージェント）の名前を使用すること。

```markdown
## 概要

<変更の目的・背景を1〜3文で説明>

## 変更内容

- <変更点1>
- <変更点2>
- <変更点3（必要に応じて）>

## 確認事項

- [ ] <テスト・動作確認事項1>
- [ ] <テスト・動作確認事項2（必要に応じて）>

🤖 Generated by [あなたのエージェント名]
```

### 3. 実行計画の提示とまとめて確認

以下を**1回の確認**でユーザーに提示する：

```
以下の内容でPull Requestを作成します。よろしいですか？

ブランチ: <branch-name> → main
タイトル: <生成したPRタイトル>

説明文:
---
<生成した説明文全文>
---
```

PRの内容をテキストで提示した後、`AskUserQuestion` ツールを使って以下の形式で確認する：

- 質問：「この内容で PR を作成しますか？」
- header：「確認」
- 選択肢：「承認して作成」（推奨）、「説明文を修正する」

ユーザーが承認した場合は手順4へ進む。修正が必要な場合はフィードバックを受けて再提示する。

### 4. PRの作成

ユーザーが承認したら以下を実行する：

1. ブランチがリモートにプッシュ済みか確認：`git ls-remote --heads origin <branch-name>`
2. リモートに存在しない場合は `git push -u origin <branch-name>` でプッシュする
3. `gh pr create` でPRを作成する（HEREDOCで説明文を渡す）：

```bash
gh pr create --title "<PRタイトル>" --body "$(cat <<'EOF'
## 概要

<概要>

## 変更内容

- <変更点>

## 確認事項

- [ ] <確認事項>

🤖 Generated by [あなたのエージェント名]
EOF
)"
```

4. 作成されたPRのURLをそのままユーザーに伝える：
   ```
   https://github.com/<owner>/<repo>/pull/<number>
   ```
   例: `https://github.com/ryo-icy/dotfiles/pull/32`

## 注意事項

- mainへの直接プッシュは行わない
- 機密情報（`.env`、`credentials`、秘密鍵など）が含まれていないことを確認する
- `--force` は使用しない（ユーザーが明示的に求めた場合を除く）
- 未コミットの変更がある場合は、先に `/commit` スキルの実行を提案する
