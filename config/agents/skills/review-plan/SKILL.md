Claudeが提案した実行プランを詳細に分析し、現在のコードベースの状態と照らし合わせて、その妥当性と安全性をレビューする。このスキルは Gemini CLI 環境での実行を前提としており、Gemini 自身の強力な分析能力とプロジェクト固有のツールを駆使して行う。

## 手順

### 1. 対象プランファイルの特定

引数でパスが指定されていない場合、以下のコマンドを使用して `~/.claude/plans/` 配下から最新のプランファイルを特定する：

```bash
ls -t ~/.claude/plans/*.md | head -n 1
```

対象ファイルが特定できない場合は、ユーザーにパスの入力を求める。

### 2. プラン内容の把握

特定したプランファイルを `read_file` で読み込み、以下の情報を抽出する：

- **目的**: 解決しようとしている課題の核心
- **変更セット**: 修正・新規作成・削除されるファイルの一覧
- **実装方針**: 提案されているロジックやアーキテクチャの変更点

### 3. コードベースとの照合と多角的な分析

プランで言及されている箇所を、Gemini のツールを用いて徹底的に調査する：

- **現状確認**: `read_file` や `grep_search` を使用し、プランが前提としているコードの状態が現在の `HEAD` と一致しているか確認。
- **依存関係の追跡**: 修正対象のシンボルが他のファイルでどのように使われているか `grep_search` で調査し、波及効果を特定。
- **規約遵守の検証**: `GEMINI.md`（および `CLAUDE.md`）に記載されたアーキテクチャ方針やコーディング規約に沿っているか確認。

### 4. 厳格なレビューの実行

以下の観点で「辛口」かつ「建設的」に評価を行う：

- **最適性**: 提案された変更はオーバーエンジニアリングではないか？より簡潔で保守性の高い代替案はないか？
- **安全性**: 境界条件の考慮漏れ、既存のバグを助長する可能性、不要な副作用がないか？
- **テスト戦略**: 変更を検証するためのテストコード（またはテスト手順）がプランに含まれているか？

### 5. 詳細なフィードバックの生成

以下の構成で、エンジニア同士のコードレビューのように具体的に提示する：

1. **レビューサマリー**: プランに対する最終評価（承認 / 修正推奨 / 却下）
2. **高く評価できる点**: プランの優れた洞察や工夫
3. **重大な懸念事項**: バグ、デグレの危険性、規約違反など（ファイル名と該当箇所を特定して記述）
4. **改善への提案**: 指摘事項を解決するための具体的な修正コード例や代替アプローチ

## 注意事項

- **推論の質**: 単なる「確認」ではなく、Gemini のツール実行結果に基づいた「批判的思考」を重視する。
- **事実ベース**: 実際のコードをツールで確認せずに「妥当」と判断してはならない。
- **一歩先のアドバイス**: プランの範囲を超えていても、プロジェクト全体の品質向上に繋がる提案（リファクタリングの機会など）があれば積極的に言及する。
